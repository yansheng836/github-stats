# 私有仓库日志脱敏方案可行性分析

> 目标：在 GitHub Actions 日志中隐藏私有仓库名称，同时支持本地解密还原，方便排查问题。

---

## 1. 现状分析

### 日志泄露位置

`query_rest()` 方法中有 3 处打印 `path`（包含 `owner/repo`）：

```python
# github_stats.py:97
print(f"[{attempt + 1}/10] {path} returned 202, retrying in {wait}s...")

# github_stats.py:117  (fallback 分支，内容相同)
print(f"[{attempt + 1}/10] {path} returned 202, retrying in {wait}s...")

# github_stats.py:125
print(f"Too many 202s for {path}, data will be incomplete.")
```

此外，异常消息中也可能包含 URL（行 105、123）：
```python
print(f"aiohttp failed for rest query: {e}")
print(f"requests also failed: {e2}")
```

### 实际日志示例

```
[1/10] repos/yansheng836/example-repo/stats/contributors returned 202, retrying in 2s...
[2/10] repos/yansheng836/example-repo/stats/contributors returned 202, retrying in 4s...
Too many 202s for repos/yansheng836/private-repo/stats/contributors, data will be incomplete.
```

其中 `example-repo` 和 `private-repo` 是私有仓库，不应暴露在公开的 Actions 日志中。

## 2. 方案设计

### 2.1 核心思路

```
Stats 调用 query_rest 时
    │
    ├─ 公有仓库 → 日志中正常显示路径
    │
    └─ 私有仓库 → 日志中显示脱敏后的路径
                   repos/yansheng836/A7X3K2/stats/contributors
                          ↑              ↑
                       正常显示      替换为加密标识
```

### 2.2 脱敏算法选择

需要满足：**确定性**（相同输入始终产生相同输出）+ **可逆性**（本地能解密）+ **无额外依赖**。

| 方案 | 依赖 | 安全性 | 确定性 | 可逆性 | 推荐 |
|------|------|--------|--------|--------|------|
| XOR + Base64 | 无（纯标准库） | 弱（仅混淆） | ✅ | ✅ | ✅ |
| AES-ECB + PKCS7 | 无（标准库 `Crypto` 需安装） | 中 | ✅ | ✅ | ❌ |
| AES-CTR（`cryptography`） | 需新增依赖 | 强 | ✅ | ✅ | ❌ |
| HMAC-SHA256（截断） | 无 | 强 | ✅ | ❌ 不可逆 | ❌ |

**推荐 XOR + Base64**：零依赖，满足日志混淆需求（非安全加密场景），确定性输出便于本地反查。

### 2.3 XOR + Base64 算法

**加密**（Actions 端）：
```
密钥 = SHA-256(盐值)[:16]       # 从 SECRET 派生 16 字节密钥
密文 = XOR(仓库名.encode(), 密钥)
标识 = Base32(密文).rstrip("=")  # Base32 仅含大写字母和数字，日志友好
```

**解密**（本地端）：
```
密钥 = SHA-256(盐值)[:16]
密文 = Base32.decode(标识 + "=" * padding)
仓库名 = XOR(密文, 密钥).decode()
```

**示例**（盐值 = `my-secret-salt`）：
```
仓库名:  example-repo
密钥:    SHA-256("my-secret-salt")[:16]
标识:    GEZDGNBVGY3TQNZTGQY======
脱敏后:  repos/yansheng836/GEZDGNBVGY3TQNZTGQY/stats/contributors

本地解密: GEZDGNBVGY3TQNZTGQY → example-repo ✅
```

### 2.4 新增环境变量

| 变量名 | 用途 | 是否必需 |
|--------|------|----------|
| `LOG_MASK_SALT` | 用于派生 XOR 密钥的盐值 | 可选（未设置则不脱敏） |

- 设置为 Repository Secret，不在日志中暴露
- 未设置时，所有仓库名称正常显示（向后兼容）

## 3. 实现要点

### 3.1 代码修改范围

| 文件 | 修改内容 |
|------|---------|
| `github_stats.py` | 新增 `_mask_repo_name()` 方法；`_fetch_contributors_for_repo()` 和 `views` 传入脱敏标识；`query_rest()` 新增 `log_prefix` 参数 |
| `generate_images.py` | 读取 `LOG_MASK_SALT`，传入 `Stats` |

### 3.2 关键改动示意

**`query_rest()` 新增 `log_prefix` 参数**：
```python
async def query_rest(self, path: str, params=None, log_prefix: str = None) -> Dict:
    display = log_prefix or path
    # 后续 print 中使用 display 替代 path
    print(f"[{attempt + 1}/10] {display} returned 202, retrying in {wait}s...")
```

**`Stats` 中脱敏逻辑**：
```python
def _mask_repo_name(self, repo: str) -> str:
    if not self._mask_key:
        return repo
    is_private = self._repo_metadata.get(repo, {}).get("isPrivate", False)
    if not is_private:
        return repo
    encrypted = bytes(a ^ b for a, b in zip(
        repo.encode(), self._mask_key * (len(repo) // len(self._mask_key) + 1)
    ))
    return base64.b32encode(encrypted).rstrip(b"=").decode()

# 调用时：
masked_path = f"repos/{repo}/stats/contributors".replace(repo, self._mask_repo_name(repo))
r = await self.queries.query_rest(path, log_prefix=masked_path)
```

### 3.3 异常消息中的 URL 泄露

异常对象 `e` 中可能包含完整 URL（如 `https://api.github.com/repos/yansheng836/example-repo/...`）。

处理方式：在 `query_rest` 的 except 块中，用 `display` 替换异常消息中的 URL：
```python
except Exception as e:
    msg = str(e).replace(path, display)
    print(f"aiohttp failed for rest query: {msg}")
```

## 4. 本地解密工具

提供一个独立的 Python 脚本，用于本地解密日志中的脱敏标识：

```python
#!/usr/bin/env python3
"""解密 Actions 日志中脱敏的私有仓库名称"""
import base64, hashlib, os, sys

SALT = os.environ.get("LOG_MASK_SALT")
if not SALT:
    print("错误：请设置环境变量 LOG_MASK_SALT（与 GitHub Secret 中的值一致）", file=sys.stderr)
    sys.exit(1)

key = hashlib.sha256(SALT.encode()).digest()[:16]

def decrypt(masked: str) -> str:
    padded = masked + "=" * (-len(masked) % 8)
    data = base64.b32decode(padded)
    return bytes(a ^ b for a, b in zip(data, key * (len(data) // len(key) + 1))).decode()

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("用法: LOG_MASK_SALT=xxx python decrypt_log.py <标识> [标识2 ...]", file=sys.stderr)
        sys.exit(1)
    for token in sys.argv[1:]:
        try:
            print(f"{token} → {decrypt(token)}")
        except Exception:
            print(f"{token} → (无法解密)", file=sys.stderr)
```

用法：
```bash
# 设置盐值（与 GitHub Secret 中的值一致）
$ export LOG_MASK_SALT="my-secret-salt"

# 解密单个标识
$ python decrypt_log.py GEZDGNBVGY3TQNZTGQY
GEZDGNBVGY3TQNZTGQY → example-repo

# 批量解密日志文件中的所有匹配项
$ grep -oP '[A-Z2-7]{8,}' actions.log | sort -u | xargs python decrypt_log.py
```

也支持一行命令：
```bash
$ LOG_MASK_SALT="my-secret-salt" python decrypt_log.py GEZDGNBVGY3TQNZTGQY
```

## 5. 潜在问题

### 5.1 XOR 安全性

- XOR + 固定密钥是弱加密，理论上可被破解
- **实际风险低**：日志中只出现短标识符（如 `GEZDGNBVGY3TQNZTGQ4DO`），攻击者需要同时知道盐值和算法才能还原
- 对于日志混淆场景已足够

### 5.2 Base32 填充符 `=`

- Base32 编码含 `=` 填充符，在日志和 URL 中可能造成混淆
- 已在编码时 `rstrip("=")` 去除，解密时补回

### 5.3 相同仓库名产生相同标识

- 确定性加密意味着同一仓库始终显示相同标识
- 这是**优点**：便于本地维护映射表
- 但也意味着攻击者如果知道一个仓库的真实名称，可以验证猜测
- 如果需要每次运行不同，可以混入时间戳（但会增加解密复杂度）

### 5.4 `views` 属性的脱敏

`views` 属性（`github_stats.py:599-615`）也调用 `query_rest`，同样需要脱敏：
```python
for repo in await self.repos:
    r = await self.queries.query_rest(f"/repos/{repo}/traffic/views")
```
需同步修改。

| 评估维度 | 结论 |

| 可行性 | ✅ 完全可行 |

| 实现复杂度 | 🟢 低（约 30 行代码） |
| 安全性 | 🟡 弱加密，但满足日志混淆需求 |

| 影响范围 | `query_rest` 的日志输出 + `Stats` 中 2 个调用点 |
